Project Title: Use of sports analytics for shot prediction in basketball¶

Data source and domain description¶

This data source has been selected from an educational competition hosted by Kaggle for the data science community. The data source was wrangled from the statistics section of the National Basketball Association (NBA) website and consists of 20 years of data collected from shots made by one of the greatest NBA players,Kobe Bryant, who recently retired from the Los Angeles Lakers in April 2016. The aim is to accurately predict the number of shots which will find the bottom of the net.This type of analysis is being frequently carried out by number of NBA franchises and teams in other sports. The results are normally used for tactical purposes to either gain further insights of perfromances of oppostion players or to track the performance of the club's own players. As basketball players get older, their performance levels tend to drop and may need to be monitored more carefully to help coaches get the best out of the player by altering their workload or playing position.

Analysis Strategy and Plans¶

The dataset contains over 20 attributes, some of which include shot type, shot distance, shot coordinates(loc_x and loc_y), latitude and longitude and a binary shot_made_flag which is the label for the dataset indication whether a shot was made or missed. Data cleaning will be carried out in R or Python. Missing values will be removed, features will be expected and removed if not required and PCA willbe carried out to investigate which features controbute most to the variance. Since this is a classification problem, the scikit-learn module in python will be employed to train a number of classification algorithms: SVM, nearest neighbours, ensemble methods using cross validation to then test the model with the best performance. The latidude/longitude and shot coordinates on the court will be used to visualise the correct shots made and shots missed on a map to gain further insights. Other visualisation plots will be used to investigate relationships between shots scored v.s teams played against, type of shot, period of the game, season and time remaining in the game. For this purpose a combination of the visualisation capabilities of Tableau and the ggplot2 package in R will be used.

Initial investigations on the data sources¶

After reading the training dataset into a python dataframe we find that there are 25 features. Missing values have been dropped from the dataset which correspond to shot IDs which need to be predicted in the test set. Columns which are redundant or categorical are then droped e.g. Team ID, Team Name, game event ID, game ID. Redundant attributes include opponent and matchup which give the same information. The total shots made (missed and scored) are plotted using the coordinate system. We find that the lat and lon (greenish yellow scatter plot) give the same information as loc_x and loc_y(bluish yellow scatter plot) so only one pair is sotred in an an array and used to plot the number of shots made (red scatter plot) which shows the majority of shots being made close to the basket, inside the semi circle and with some shots made along specific areas along the three point line. We then carry out Principal Component Analysis to investigate which of the 5 numerical features account for the most variance. The bar chart showed that the % variance ratio was around 20% for most of the features showing that all features contribute almost equually to the variance. This is further validated through the scatter plot of principal component 1 and 2, which shows a rougly speherical distribution in space with the points slightly most distributed along PC1 as expected. Hence all 5 features will be used for training models described aboive.

NBA_train['season'] = NBA_train['season'].map(lambda x: x[0:4]).astype(np.int64) # Converting the year range in the season column to a single year by stripping character following hyphen 


train_PCA = NBA_train.values  #Preparing data for PCA : we need to convert data frame into array  

train_PCA = train_PCA[:,[4,5,6,7,8]] # extracting non categorical columns from array as PCA only recognises numbers. Alse excluded the labels (shot_made_flag) 

pca = PCA(n_components=5) # the components are mins remaining, period, season,seconds remaining and shot distance 

train_PCA_var = pca.fit_transform(scale(train_PCA)) # scaling the columns so they all have the same range before PCA and then applying the transformation 

varianceratio = pca.explained_variance_ratio_
print(' ')
print(pca.explained_variance_ratio_) # prints array with componets with highest variance ratio in descending order
print(' ')

train_PCA_cumsum_var = np.cumsum(np.round(pca.explained_variance_ratio_, decimals=4)*100)

print(' ')
print(train_PCA_cumsum_var) # prints array with cummulative variance for the 5 components
